Section IV · Recommendations
04 / 04

Fix before next time.

Six concrete improvements identified across content, policy, and operations. Each is scoped to be addressable before the next event in the same series.

01

Add a pre-event "what to bring" message to all confirmed attendees.

Dress code (38 messages) and prohibited items (in 19 of the "other" bucket) account for nearly 20% of inbound, much of it answerable with a single proactive push 48 hours before the event. Estimated reduction in event-week support load: 12 to 16%.

02

Clarify the parking-to-entrance walk in pre-event comms.

Parking questions (19 messages) were almost entirely about distance and walkability, not availability. A short note in the event-page logistics section, plus a map link in the confirmation email, would address the majority.

03

Pre-empt the "I bought the wrong ticket" path with a clearer category description at checkout.

Five refund cases this event traced to ticket-type confusion, all of which converted to paid upgrades. Better checkout copy could prevent the confusion entirely and reduce upgrade-pressure load on the team.

04

Add a payment-recovery flow for the 14 incomplete-payment cases.

Payment-failed and payment-pending orders sat unrecovered for the event window. Estimated recoverable revenue at observed conversion rates: AED 3,200 to 5,400 per event of this size. This is the natural v1.1 extension of the deflection system.

05

Document the medical-exception policy explicitly in the published refund policy.

Two medical refund requests this event were handled correctly through escalation, but the published policy is ambiguous on what documentation is required. A clear policy paragraph and an internal SOP would reduce decision latency.

06

Bilingualise three KB sections that produced low-confidence Arabic responses.

Sections on backstage inclusions, table reservations, and the medical exception flow were the three cases where the Arabic generation confidence dropped below threshold and triggered escalation. Native rewrite, not translation, will close this gap.

Bottom Line

One event. AED 9,750 protected. 19 hours of team time saved. Six clear improvements queued for the next event. The system paid for the pilot inside the event window.
